# 邮箱域名限制功能更新

## 更新内容

将邮箱域名限制输入方式从**手动输入**改为**从账号池中智能选择**，提升用户体验。

## 主要改进

### 1. 后端 API 新增
- **新接口**: `GET /admin/accounts/email-domains`
- **功能**: 从账号池中提取所有邮箱域名，统计每个域名的可用账号数和总账号数
- **实现位置**: `handler/admin.go` - `AccountEmailDomains` 函数
- **路由配置**: `main.go` 第 128 行

### 2. 前端界面改造
#### 2.1 输入方式变更
- **原方式**: 文本输入框，手动输入域名（如 `gmail.com,outlook.com`）
- **新方式**: 多选下拉菜单，从账号池实际域名中选择

#### 2.2 交互优化
- 默认显示"不限制"
- 支持多选，每个域名旁显示可用账号数
- 实时更新显示选中的域名数量（如 `gmail.com +2`）
- 选择后立即更新生成提示信息

### 3. 前端实现细节
**修改文件**: 
- `static/index.html`: 将 `<input>` 改为 `<div class="k-dropdown">`
- `static/js/cards.js`: 
  - 新增 `genEmailDomains` 数组存储选中域名
  - 新增 `loadEmailDomains()` 加载域名列表
  - 新增 `toggleEmailDomain()` 切换域名选择
  - 新增 `updateEmailDomainsDisplay()` 更新显示文本
  - 新增 `updateEmailDomainsChecks()` 更新选中标记

## 使用说明

### 生成卡密时选择域名限制

1. 点击"生成卡密"按钮
2. 在"邮箱域名限制"下拉菜单中：
   - 选择"不限制"：清空所有选择，卡密可分配任何域名的账号
   - 点击具体域名（如 `gmail.com (5 可用)`）：添加到限制列表
   - 再次点击已选域名：取消选择
3. 可以选择多个域名，如同时选择 `gmail.com` 和 `outlook.com`
4. 生成提示会实时显示：`将生成 1 张卡密，每张绑定 1 个账号，账号订阅：Pro，限制域名：gmail.com, outlook.com。`

### 域名显示规则

- 0 个选中：显示"不限制"
- 1 个选中：显示域名（如 `gmail.com`）
- 多个选中：显示第一个+数量（如 `gmail.com +2`）

### 域名排序规则

1. 按可用账号数降序
2. 按总账号数降序
3. 按域名字母顺序

## 技术实现

### 域名提取逻辑
```go
// 从账号邮箱中提取域名
email := "user@gmail.com"
domain := "gmail.com" // 提取 @ 后面的部分
```

### 多选交互
- 使用复选标记（✓）显示选中状态
- 不自动关闭下拉菜单，方便连续选择
- 点击外部自动关闭（与现有下拉菜单行为一致）

## 数据流

1. **打开生成卡密对话框** → 调用 `/admin/accounts/email-domains` API
2. **后端查询** → 从 `accounts` 表提取 `email` 字段，解析域名并统计
3. **前端展示** → 显示域名列表，标注可用账号数
4. **用户选择** → 更新 `genEmailDomains` 数组
5. **生成卡密** → 将选中域名用逗号连接发送到后端

## 优势

✅ **智能选择**: 只显示账号池中实际存在的域名，避免输入错误  
✅ **实时统计**: 显示每个域名的可用账号数，辅助决策  
✅ **多选支持**: 可同时限制多个域名  
✅ **用户友好**: 下拉选择比手动输入更直观便捷  
✅ **零学习成本**: 与现有订阅选择器交互一致

## 兼容性

完全兼容之前实现的后端逻辑，仅改变前端输入方式，不影响：
- 卡密数据库结构
- 账号分配逻辑
- 域名匹配算法
