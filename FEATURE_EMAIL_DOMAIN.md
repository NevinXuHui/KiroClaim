# 卡密邮箱域名限制功能

## 功能概述

本功能允许管理员在生成卡密时指定邮箱域名限制，确保该卡密只能分配给特定邮箱域名的账号。

## 实现原理

1. **数据模型**：在 `Card` 模型中添加 `AllowedEmailDomains` 字段，存储允许的邮箱域名列表（逗号分隔）
2. **生成卡密**：管理员可以在生成卡密时选填邮箱域名限制（如 `gmail.com,outlook.com`）
3. **账号分配**：激活/兑换卡密时，系统会从账号池中筛选符合邮箱域名限制的账号
4. **域名匹配**：系统会检查账号的邮箱后缀是否与卡密允许的域名匹配

## 使用说明

### 生成带域名限制的卡密

1. 进入管理后台的"卡密管理"页面
2. 点击"生成卡密"按钮
3. 填写基本信息：
   - 生成数量
   - 账号数量
   - 账号订阅
4. **可选**：在"邮箱域名限制"字段填写允许的域名
   - 单个域名：`gmail.com`
   - 多个域名：`gmail.com,outlook.com,yahoo.com`
   - 留空：不限制，可分配任何邮箱的账号
5. 点击"立即生成"

### 邮箱域名格式说明

- **支持格式**：
  - 不带 @ 符号：`gmail.com`
  - 带 @ 符号：`@gmail.com`（系统会自动去除）
- **多个域名**：用英文逗号 `,` 分隔
- **大小写**：不敏感，系统会自动转换为小写
- **空格**：会自动去除多余空格

### 示例

#### 示例 1：限制为 Gmail 邮箱
```
邮箱域名限制：gmail.com
```
该卡密只能分配邮箱为 `xxx@gmail.com` 的账号

#### 示例 2：限制为多个邮箱域名
```
邮箱域名限制：gmail.com, outlook.com, hotmail.com
```
该卡密可以分配邮箱为：
- `xxx@gmail.com`
- `xxx@outlook.com`
- `xxx@hotmail.com`

#### 示例 3：不限制
```
邮箱域名限制：（留空）
```
该卡密可以分配任何邮箱的账号

## 卡密列表显示

在卡密列表中，有邮箱域名限制的卡密会显示一个黄色的锁图标徽章，鼠标悬停可查看完整的域名列表。

## 技术实现

### 数据库字段

```go
type Card struct {
    // ... 其他字段
    AllowedEmailDomains string `gorm:"type:varchar(500);default:''"` 
    // 允许的邮箱域名，逗号分隔，如 "gmail.com,outlook.com"
}
```

### API 接口

#### 生成卡密

**请求**：
```json
POST /admin/cards/generate
{
  "count": 10,
  "account_count": 1,
  "subscription": "pro",
  "allowed_email_domains": "gmail.com,outlook.com"
}
```

**字段说明**：
- `allowed_email_domains`：可选字段，允许的邮箱域名（逗号分隔）

### 核心函数

#### 1. normalizeEmailDomains
规范化邮箱域名列表：
- 去除空白
- 转换为小写
- 去除 @ 符号
- 去重

#### 2. emailMatchesDomains
检查邮箱是否匹配允许的域名列表：
- 如果 `allowedDomains` 为空，返回 `true`（不限制）
- 检查邮箱后缀是否在允许的域名列表中

#### 3. isDispatchable
判断账号是否可分配（已增加邮箱域名检查）：
- 账号状态检查
- 订阅匹配检查
- **邮箱域名匹配检查**（新增）

## 注意事项

1. **数据库迁移**：首次运行需要数据库自动迁移，添加 `allowed_email_domains` 字段
2. **账号池充足**：确保账号池中有足够多符合域名限制的账号
3. **域名格式**：建议使用标准域名格式，不含 @ 符号
4. **性能影响**：邮箱域名过滤会稍微增加账号筛选时间，但影响很小

## 测试建议

1. **无限制测试**：生成卡密时不填写域名限制，验证可以正常分配账号
2. **单域名限制测试**：生成卡密时填写单个域名，验证只能分配对应域名的账号
3. **多域名限制测试**：生成卡密时填写多个域名，验证可以分配这些域名的账号
4. **账号不足测试**：生成域名限制的卡密，但账号池没有对应域名的账号，验证错误提示
5. **前端显示测试**：验证卡密列表正确显示域名限制徽章

## 常见问题

### Q: 如果账号池中没有符合域名限制的账号会怎样？
A: 激活卡密时会返回错误："账号不足，请联系管理员补充"或"账号池账号不足"

### Q: 可以修改已生成卡密的域名限制吗？
A: 当前版本不支持修改，需要删除旧卡密并重新生成

### Q: 域名限制是否区分大小写？
A: 不区分，系统会自动转换为小写进行匹配

### Q: 能否支持子域名限制？
A: 当前版本只支持精确域名匹配，不支持通配符或子域名匹配
